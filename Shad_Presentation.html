Fundamentals

What is a PCAP File?

A binary recording of raw data packets flowing through a USB bus. In CTFs, flags are hidden inside captured traffic waiting to be decoded.

PCAP = Packet Capture

Records every byte transferred between host and device. Each packet contains raw HID report data — keystrokes, mouse deltas, or bulk transfer payloads.

Tools to Analyze PCAPs

  • Wireshark — graphical viewer with USB dissectors
  • tshark — command-line Wireshark for scripting
  • scapy — Python library used in this tool
  • tcpdump — low-level capture and filtering

CTF Challenges

Types of USB CTF Challenges

Four distinct categories, each requiring a different decoding approach.

⌨️

Keyboard

Captures every keystroke. Decode modifier keys, Shift, CapsLock, and Backspace to reconstruct what was typed and extract the hidden flag.

→ Text / Flag string
🖱️

Mouse

Track X/Y deltas to plot cursor movement. Left-click drag paths often spell out letters or a complete flag when rendered as a PNG image.

→ PNG image / Drawing
🖨️

Printer

Extract bulk transfer data from print jobs. Output may be PCL, PostScript, or PDF — renderable documents containing the hidden flag.

→ PCL / PostScript / PDF
💾

Mass Storage

Carve known file types from raw USB bulk transfer data using magic byte signatures — no filesystem required.

→ PNG / ZIP / ELF / etc.

// TRACKING LOGIC
x, y = 2000, 2000  # center 4000×4000

for raw in data_list:
  buttons = raw[0]
  dx = signed_byte(raw[1])  # ±127
  dy = signed_byte(raw[2])  # ±127

  x = clamp(x + dx, 0, 4000)
  y = clamp(y + dy, 0, 4000)

  if buttons & 0x01:  # left button held
    click_points.append((x, y))

Why 3 Output Images?

Different CTF challenges hide flags differently. Sometimes only the connected click-path reveals text. Other times, dot clusters form a pattern. Generating all three maximizes your chance of seeing the flag.

Signed Byte Conversion

Mouse deltas are signed — values above 127 mean negative movement (left/up). Helper: val - 256 if val > 127 else val

Pro Tips

CTF Tips & Tricks

Hard-earned strategies for each challenge type. Always start with --mode hex to study packet patterns first.

⌨️ Keyboard Tips

If output is garbled, check initial CapsLock state — it may have been toggled before capture began
Look for [CTRL+V] — the flag may have been pasted, not typed character by character
Many backspaces? Read the keylog in reverse — the real input may be obscured by corrections
Use grep -i 'flag\|ctf\|{' keyboard_output.txt to search the saved output

🖱️ Mouse Tips

Open all 3 PNG files — each reveals different visual information about the movement
Image too small? Adjust start coordinates from (2000,2000) to match actual origin
Check right-click-only paths too — some challenges use right-click to draw the flag
Canvas is 4000×4000 px — zoom in heavily on the click region in your image viewer

🖨️ Printer Tips

Open .ps PostScript files in a text editor — they may contain plaintext flag directly
PCL files often have readable ASCII between binary sections — worth scanning
Render with GhostScript: gs -sDEVICE=png16m -o out.png file.ps
The tool automatically runs a strings search and prints any readable matches found

💾 Storage Tips

Run strings storage_raw.bin immediately — flags often appear in plaintext
Use binwalk storage_raw.bin for nested or compressed file detection
Password-protected ZIPs? Try the rockyou.txt wordlist with fcrackzip or john
Always start with --mode hex to study raw byte patterns before choosing a mode
