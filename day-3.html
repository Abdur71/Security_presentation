PHASE 02
DECOMPILE

Java Source Code Review

CRITICAL Hardcoded AES Key
Encryption key embedded directly in CryptoClass source — extractable by anyone with basic reverse engineering knowledge.
// CryptoClass.java — exposed after decompile private static final String KEY = "This is the Key";
Impact: All stored credentials trivially decryptable.
CRITICAL Static IV — AES/CBC
AES-CBC mode requires a unique random IV per operation. Static IV enables pattern leakage and known-plaintext attacks.
// Same IV used for EVERY encryption IvParameterSpec iv = new IvParameterSpec(STATIC_IV); // Identical plaintext → identical ciphertext
Impact: Confidentiality completely broken.
MEDIUM WebView — JS Enabled
JavaScript enabled in ViewStatement WebView. Without addJavascriptInterface, XSS risk remains if untrusted content is loaded.
webView.getSettings() .setJavaScriptEnabled(true);
Impact: XSS attack surface in statement viewer.
HIGH No SSL Pinning
Searched for TrustManager, HostnameVerifier, CertificatePinner, OkHttpClient — none found. Default system trust store only.
// grep result: 0 matches grep -R "CertificatePinner" ./ grep -R "TrustManager" ./ ← NO RESULTS
Impact: Full MITM vulnerability on any network.

Hardcoded Encryption KeyStore keys in Android Keystore System — use key derivation (PBKDF2 or HKDF)
P1
Static Initialization VectorGenerate cryptographically random IV per encryption operation (SecureRandom)
P1
Client-Side AuthenticationMove ALL authentication validation server-side — never trust client logic
P1
SMS Password TransmissionRemove BroadcastReceiver or add strict permission + explicit user confirmation
P1
Exported Banking ActivitiesSet android:exported="false" or enforce android:permission on each component

No Tamper DetectionImplement Play Integrity API attestation for device and APK integrity
P2
