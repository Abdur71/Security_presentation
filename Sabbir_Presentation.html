What Is a Capture The Flag Competition?

Introduction
A Capture The Flag (CTF) competition is a cybersecurity event where participants solve security challenges to discover hidden strings called flag{...} — submitted to a scoring platform for points. CTFs bridge the gap between academic theory and real-world offensive security skills, and are held globally by universities and professional organisations.
📋
Jeopardy-Style
Category board with point values. Teams choose freely which challenges to attempt. Most common for online competitions worldwide.
⚔️
Attack–Defense
Teams defend their own service while simultaneously exploiting identical services run by opponents. Closest to real-world scenarios.
🔓
Boot2Root / HTB
Compromise a target machine from zero access to root, collecting flags along the privilege escalation path.
50+
Major CTF events per year globally
10k+

Reverse Engineering — 4-Phase Methodology

REV
01
Triage
file ./bin — format & arch
strings — hardcoded data
binwalk -E — entropy
ldd ./bin — linked libs
Goal: understand in <5 min
02
Static Analysis
Load in Ghidra / IDA
Navigate to main()
Rename vars + functions
Trace input → compare
Goal: understand the check
03
Dynamic Analysis
GDB + GEF breakpoints
break strcmp; inspect regs
x/s $rdi / $rsi
ltrace / strace
Goal: see runtime values
04
Reconstruct
Python — invert algorithm
angr — symbolic exec
Z3 — constraint solver
Verify → test → submit
Goal: recover the flag
🧭 Core Principle: Always work backwards from the success condition — find what the binary accepts, then reverse-engineer the path to produce that exact input.

Reverse Engineering — Tools Arsenal

REV
Ghidra
Decompiler / Disassembler
NSA's open-source RE framework. Decompiles to C pseudocode, supports 50+ architectures, Java/Python scripting API. Free.
IDA Pro
Disassembler
Industry gold standard. Hex-Rays decompiler. Best for complex obfuscated binaries. Commercial but ubiquitous in professional work.
GDB + GEF
Debugger
GNU Debugger with GDB Enhanced Features plugin. Heap visualisation, colour output, ROP helpers, De Bruijn pattern analysis.
radare2
Analysis Suite
Powerful CLI framework (r2) with Cutter GUI frontend. Fully free and open-source. Extensive scripting and automation support.
angr
Symbolic Execution
Python framework for automated binary analysis. Explores execution paths symbolically and solves for inputs that reach a target state.
Z3
SMT Solver
Microsoft's SMT solver. Directly solves mathematical constraints to recover flag characters without brute force. Used with angr or standalone.
ltrace / strace
Dynamic Tracing
Trace library calls (ltrace) and system calls (strace) at runtime. Invaluable for rapid triage and understanding I/O behaviour instantly.
binwalk
Firmware Analysis
Detect and extract embedded files, compressed sections, and encrypted regions. Entropy analysis identifies packed or encrypted code.

REV Example — "SecureVault"  250 pts

Worked Example
1
File Triage
# Initial identification
$ file ./securevault
ELF 64-bit, dynamically linked

$ strings ./sv | grep -iE 'flag|grant'
Access Granted! Flag: %s
Wrong code. Try again.

# Flag computed at runtime
# → must reverse the check
2
Ghidra Decompile
// validate_code() — renamed
int validate_code(char *input) {
  char key = 0x37;
  char enc[] = { 0x52,0x44,
                0x49,0x4d... };
  for (i=0; i<8; i++)
    expected[i] = enc[i] ^ key;
  return strcmp(input, expected);
}
3
Python Solve Script
#!/usr/bin/env python3
# XOR is self-inverse

encoded = [0x52,0x44,0x49,
           0x4d,0x4e,0x55,
           0x4e,0x46]
key = 0x37

secret = ''.join(
  chr(b ^ key) for b in encoded)

print(secret) # → 'epicctf1'
✓
Flag Captured
$ ./securevault
Enter the secret code: epicctf1

Access Granted!

flag{x0r_15_n0t_
   3ncrypt10n_l0l}

⏱ Solve time: ~20 min
📌 XOR is not encryption

Digital Forensics — Network PCAP Analysis

DFIR · PCAP
1
Assess Protocol Mix
Statistics → Protocol Hierarchy shows composition
tshark -r cap.pcap -qz io,phs
2
Apply Display Filters
Isolate relevant protocol traffic before inspection
http.request.method == "POST"
3
Follow Streams
Reconstruct full TCP/HTTP conversations as text
tshark -z follow,tcp,ascii,0 -q
4
Export Objects
File → Export Objects → HTTP/SMB/FTP/TFTP
--export-objects http,./out/
5
Check DNS Exfiltration
Encoded subdomains signal covert data transfer
-Y dns -T fields -e dns.qry.name | sort
● tshark cheatsheet
# Protocol breakdown
tshark -r cap.pcap -qz io,phs

# Export HTTP objects
tshark -r cap.pcap --export-objects http,./out/

# Follow TCP stream 0
tshark -r cap.pcap -z follow,tcp,ascii,0 -q

# DNS query names (exfil detection)
tshark -r cap.pcap -Y dns \
  -T fields -e dns.qry.name | sort | uniq

# HTTP POST data extraction
tshark -r cap.pcap \
  -Y 'http.request.method==POST' \
  -T fields -e http.file_data

# FTP credentials
tshark -r cap.pcap -Y ftp \
  -T fields -e ftp.request.arg

# Identify partitions
fdisk -l image.img

# Mount read-only

# List all + deleted files
fls -r -d image.img

# Recover by inode number
icat image.img 42 > recovered_file

# Carve by file signature
photorec image.img
foremost -i image.img -o ./out/

vol -f mem.raw windows.info

vol -f mem.raw windows.cmdline
vol -f mem.raw windows.cmdscan

  --pid 1234 --dump-dir ./dumps/

# String search in process memory
strings proc.dmp | grep -i flag

DFIR Example — "LostPackets"  150 pts

Worked Example
ChallengeAn employee was caught exfiltrating sensitive data. Network traffic was captured. Identify what was stolen. [Attachment: capture.pcap]
1
PCAP Triage
1,423 packets · 42s · Protocol split: 68% HTTP, 22% DNS, 10% TCP handshakes.
capinfos capture.pcap
tshark -r cap.pcap -qz io,phs
2
HTTP Object Export
HTTP dominates → exporter reveals upload_20231102.b64
tshark -r cap.pcap --export-objects http,./out/
3
Decode Payload
The .b64 extension immediately signals base64 encoding. Pipe directly to decoder.
cat upload.b64 | base64 -d
4
DNS Due Diligence
Always check DNS for covert channels regardless of already having a flag.
tshark -Y dns -T fields -e dns.qry.name
→ No suspicious subdomains found ✓
# The extracted file content
$ cat ./out/upload_20231102.b64
ZmxhZ3twY2FwXzRuNGx5czFzX3IzdjM0bHNfNGxsfQ==

# Decode from base64
$ cat upload_20231102.b64 | base64 -d
🏁   Flag Captured   🏁
flag{pcap_4n4lys1s_r3v34ls_4ll}
Lesson LearnedThe .b64 extension was a hint hiding in plain sight. Read filenames carefully before opening tools. Always run a DNS exfiltration check even after finding a flag — thoroughness is a professional standard.

The CTF Problem-Solving Mindset

Metacognition
🔍
Adversarial Curiosity
Assume every system has a flaw. Your job is to find it. This mindset is the foundation of all security research, not just CTFs.
🧪
Hypothesis-Driven
Form explicit, testable theories before applying tools. "I think the flag is LSB-encoded" is testable. Random tool-clicking is not.
📝
Document Everything
Keep a live notepad throughout every challenge. The flag is often found by connecting observations made hours apart.
⏱️
Know When to Pivot
30 minutes with zero new information means change approach. Spending 4 hours on a 50pt challenge ignoring 400pt ones is a strategic error.
🤝
Collaborate Openly
Brief teammates when handing off. Explaining a problem aloud — even to yourself — frequently surfaces the solution instantly.
🔄
Iterate Relentlessly
Every failed attempt narrows the solution space. Failure is information. Record what failed before moving to the next hypothesis.

Common Mistakes & How to Avoid Them

Anti-Patterns
✗Ignoring file metadata and properties
→
✓Always run exiftool + binwalk + strings before any specialist tool
✗Not reading the challenge description carefully
→
✓Re-read 3× before starting tools. "Deleted", "hidden", "recovered" are precise, intentional author hints
✗Jumping to Volatility, angr, or Ghidra first
→
✓Begin with file, strings, exiftool. Complex tools come after basic triage yields nothing
✗Working alone in isolation for more than 30 minutes
→
✓Explain the problem to a teammate or rubber-duck debug. Fresh perspective reliably unblocks progress
✗Not documenting negative results and dead ends
→
✓Record every failed attempt — it narrows the space and prevents repeating the same dead end under pressure
✗Assuming a candidate string is already plaintext
→
✓Flags may be base64, hex, reversed, compressed, or split. Always decode before dismissing any result
